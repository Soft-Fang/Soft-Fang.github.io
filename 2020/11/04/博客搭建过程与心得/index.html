第一步：安装Git,Node.js（环境配置），hexo

Node.js安装以及环境配置

安装node.js后

在nodejs文件夹下，创建两个为文件夹： node_global;node_cecal

这是用来放安装过程的缓存文件以及最终的模块配置位置

使用下面命令(cmd)将npm的全局模块目录和缓存目录配置到我们刚才创建的那两个目录

随后在此电脑属性中的高级系统设置里设置环境变量

在系统变量里添加

变量名(N):NODE-PATH

变量值(V):"node_modules文件的路径"

在Path变量名中添加“node_global的文件路径”

安装完成后在除C盘外任意一个盘中新建一个blog文件夹，然后在文件夹中右键打开git bash，输入

依然用hexo -v查看版本

随后初始化hexo(命令要在执行完成后再输入下一条)

随后在myblog文件夹中就会有

  • node_modules: 依赖包
  • public：存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题
  • ** _config.yml: 博客的配置文件**

按住ctrl+c就可以关闭服务

第三步：部署SSH以及hexo

SSH

在刚才的git bash中

yourname/email都是刚才GitHub注册的

可以用这两条检查你有没有输对

然后创建SSH，一路回车就行

可以在ssh-keygen之后加上-f 指定文件输出 注意输入空格 可以指定到 C:\Users\(用户名)\.ssh

ssh，简单来讲，就是一个秘钥，其中，id_rsa是这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当链接到自己GitHub的账户时，它就会根据公钥匹配私钥，当能够相互匹配时，才能够顺利的通过git上传文件到GitHub上。

而后在GitHub的setting中，找到SSH keys的设置选项，点击New SSH key
把id_rsa.pub里面的信息复制进去。检测 ↓

然后

hexo clean清除了你之前生成的东西，也可以不加。
hexo generate 顾名思义，生成静态文章，可以用 hexo g缩写
hexo deploy 部署文章，可以用hexo d缩写
